# Setup

## 1. Profile README
1. Create a new repo named exactly `srouelle` (must match your username).
2. Add `README.md` to it — this is the only file that repo needs.
3. It will automatically render at the top of https://github.com/srouelle

## 2. Portfolio site (GitHub Pages)
1. Create a new repo named `srouelle.github.io` (must match this pattern exactly).
2. Upload `index.html`, `project1.html`, `project2.html`, `project3.html`, `skills.html`, and `style.css` to the root of that repo.
3. Go to the repo's **Settings → Pages**, set source to the `main` branch, root folder.
4. Your site will be live at `https://srouelle.github.io` within a few minutes.
5. Update the "Full portfolio" link in `README.md` once it's live (it already points to the right URL).

## 3. Before publishing, replace the placeholders
- Bio text on `index.html`
- LinkedIn URL and email (appears in nav footer of every page + README)
- Project names, summaries, tech tags, repo/demo links, and problem/approach/results text on each project page
- Skill tags on `skills.html` if they don't match your actual stack
